# Entry 015 Runtime Work Is Still The Part That Fights Me

One thing I’ve learned the hard way building Addison is that runtime work is still the part that fights me the most.

There’s a certain kind of pain to it that is hard to explain if you haven’t lived in it for a while.

It’s not the clean kind of work.
It’s not the kind that usually gives you a neat before-and-after.
It’s not even always the kind of work that feels satisfying when you’re in the middle of it.

A lot of the time it feels like friction.

You fix one thing and expose two more.
You get one seam behaving and another one starts showing its edge.
You prove something in bounded form, but living with it in actual use still reveals where the roughness is.
And when you’re doing it solo, there isn’t really anyone to hand it off to.
At some point you just have to keep pushing until the thing holds together well enough to earn the next layer.

That has definitely been true on the shell side.

It’s in a better place now than it was.
That part is real.

The shell is active and operational now, even if it’s still not my primary runtime in practice. The broader embodiment line landed well enough that I can say it works for what I need it to do right now. The avatar side is no longer living in the category of vague ambition. It is there. It is functioning. It is usable.

But usable and effortless are not the same thing.

And that distinction matters to me because I don’t want to romanticize this part of the build.

A lot of the shell and runtime work has been me muscling through surfaces that are just harder for me. Presentation surfaces. runtime behavior. synchronization. edge-case truth. the strange space where architecture, user experience, and live behavior all start touching each other. That’s not the part of the work that comes most naturally to me, but it still has to be done if I want Addison to become what I actually see in my head.

So I do it.

Not because I enjoy every second of it.
Not because it’s the cleanest line in the project.
But because being solo means there are parts you love and parts you carry anyway.

That’s what this stretch has felt like.

Carrying it anyway.

And to be fair, that effort is finally turning into something visible.

The shell is in a better place.
The embodiment line is in a better place.
The current avatar targets are landing well enough for their actual purpose.
There are still bugs.
There are still rough edges.
There are still things I want to correct.

But it is no longer one of those areas where I have to speak mostly in future tense.

It is here now.

And even if runtime work still feels like the part that fights me the most, I think I’m getting better at fighting through it without losing the thread.
